---
title: "Password Authentication"
description: "Authenticate using username and password via HTTP Basic Auth"
---

Password authentication is the standard method using HTTP Basic Auth.

## Format

```
{username}-{parameters}:{password}@{host}:{port}
```

## Credential Structure

| Component | Format | Length | Example |
|-----------|--------|--------|---------|
| Username | `sub_` + random | 12 chars | `sub_k8mX2pL9` |
| Password | random | 16 chars | `Qw4nR7vB3xY9mK2p` |

The username prefix `sub_` indicates a subscription credential.

## Usage Examples

### Basic Request

```bash
curl -x "http://sub_k8mX2pL9:Qw4nR7vB3xY9mK2p@proxy.turnoxy.com:1318" \
     https://example.com
```

### With Parameters

Parameters are appended to the username using `-` as delimiter:

```bash
curl -x "http://sub_k8mX2pL9-country-US:Qw4nR7vB3xY9mK2p@proxy.turnoxy.com:1318" \
     https://example.com
```

### Multiple Parameters

```bash
curl -x "http://sub_k8mX2pL9-pool-dc-country-US-session-abc123:Qw4nR7vB3xY9mK2p@proxy.turnoxy.com:1318" \
     https://example.com
```

## Authentication Errors

All authentication failures return HTTP `407 Proxy Authentication Required`:

- Invalid username
- Wrong password
- Expired subscription
- Exhausted traffic quota

<Note>
For security reasons, the specific failure cause is not disclosed in the response.
</Note>
