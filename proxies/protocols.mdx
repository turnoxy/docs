Turnoxy supports three proxy protocols to cover all use cases.

## Supported Protocols

| Protocol | Port | Use Case |
|----------|------|----------|
| HTTP | 1318 | HTTP requests |
| HTTPS | 1318 | HTTPS requests (CONNECT tunnel) |
| SOCKS5 | 1319 | Any TCP traffic |

## HTTP Proxy

For plain HTTP requests:

```bash
curl -x "http://sub_k8mX2pL9:Qw4nR7vB3xY9mK2p@proxy.turnoxy.com:1318" \
     http://example.com
```

## HTTPS Proxy

For HTTPS requests, the proxy creates a CONNECT tunnel:

```bash
curl -x "http://sub_k8mX2pL9:Qw4nR7vB3xY9mK2p@proxy.turnoxy.com:1318" \
     https://example.com
```

<Note>
The proxy URL uses `http://` even when accessing HTTPS sites. The proxy establishes an encrypted tunnel to the target.
</Note>

### How HTTPS Tunneling Works

```
Client ──CONNECT──→ Proxy ──TCP──→ Target
         │              │
         │    ══════════════════    Encrypted tunnel
         │              │           Proxy cannot see content
         └──────────────┴── HTTP request/response
```

The proxy only sees:
- Target hostname and port
- TCP connection status

The proxy cannot see:
- Request/response headers
- Request/response body
- HTTP status codes

## SOCKS5 Proxy

For any TCP traffic (not just HTTP):

```bash
curl -x "socks5://sub_k8mX2pL9:Qw4nR7vB3xY9mK2p@proxy.turnoxy.com:1319" \
     https://example.com
```

### SOCKS5 Use Cases

- Non-HTTP protocols (FTP, SMTP, etc.)
- Applications that don't support HTTP proxies
- Lower-level network tools

## Protocol Comparison

| Feature | HTTP/HTTPS | SOCKS5 |
|---------|------------|--------|
| Port | 1318 | 1319 |
| HTTP Support | Yes | Yes |
| Non-HTTP Support | No | Yes |
| Authentication | Basic Auth | Username/Password |
| Parameters | Full support | Full support |
