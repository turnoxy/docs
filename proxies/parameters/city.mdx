---
title: "city"
description: "Target specific cities within a country"
---

The `city` parameter targets IPs from a specific city.

## Specification

| Property | Value |
|----------|-------|
| Type | String |
| Format | Lowercase + underscore |
| Characters | `[a-z_]` |
| Case | Insensitive (stored as lowercase) |
| Requires | `country` parameter |

## Naming Convention

- Use English names from GeoNames
- Replace spaces with underscores
- All lowercase

| Original | Format |
|----------|--------|
| New York | `new_york` |
| Los Angeles | `los_angeles` |
| San Francisco | `san_francisco` |

## Usage

```bash
# New York City, US
sub_k8mX2pL9-country-US-city-new_york:pass@proxy.turnoxy.com:1318

# Los Angeles, US
sub_k8mX2pL9-country-US-city-los_angeles:pass@proxy.turnoxy.com:1318

# London, UK
sub_k8mX2pL9-country-GB-city-london:pass@proxy.turnoxy.com:1318

# Tokyo, Japan
sub_k8mX2pL9-country-JP-city-tokyo:pass@proxy.turnoxy.com:1318
```

## With State Parameter

For maximum precision, combine `city` with `state`:

```bash
# Los Angeles, California, US
sub_k8mX2pL9-country-US-state-california-city-los_angeles:pass@proxy.turnoxy.com:1318
```

This is useful when multiple cities share the same name across different states.

## Geolocation Hierarchy

```
country (required)
    │
    ├── state (optional)
    │
    └── city (optional, can be used with or without state)
```

### Valid Combinations

| Combination | Example |
|-------------|---------|
| country only | `country-US` |
| country + state | `country-US-state-california` |
| country + city | `country-US-city-los_angeles` |
| country + state + city | `country-US-state-california-city-los_angeles` |

### Invalid Combinations

| Combination | Error |
|-------------|-------|
| state only | `STATE_REQUIRES_COUNTRY` |
| city only | `CITY_REQUIRES_COUNTRY` |

## Errors

| Error Code | HTTP | Cause |
|------------|------|-------|
| `CITY_REQUIRES_COUNTRY` | 400 | `city` used without `country` |
| `INVALID_CITY` | 400 | City name not recognized |
