---
title: "pool"
description: "Select IP pool type: residential, datacenter, ISP, or mobile"
---

The `pool` parameter selects which type of IP addresses to use.

## Specification

| Property | Value |
|----------|-------|
| Type | Enum |
| Values | `res`, `dc`, `isp`, `mob` |
| Default | `res` |
| Case | Insensitive |

## Pool Types

| Value | Full Name | Characteristics | Best For |
|-------|-----------|-----------------|----------|
| `res` | Residential | Real home IPs, high quality, slower | Social media, e-commerce, high anonymity |
| `dc` | Datacenter | Fast, large pool, easier to detect | High-volume scraping, speed priority |
| `isp` | ISP | Fast, high quality, smaller pool | Long-term tasks, balanced needs |
| `mob` | Mobile | Carrier IPs, dynamic, highest anonymity | Mobile simulation, highest security |

## Usage

```bash
# Residential (default)
sub_k8mX2pL9-pool-res:pass@proxy.turnoxy.com:1318

# Datacenter
sub_k8mX2pL9-pool-dc:pass@proxy.turnoxy.com:1318

# ISP
sub_k8mX2pL9-pool-isp:pass@proxy.turnoxy.com:1318

# Mobile
sub_k8mX2pL9-pool-mob:pass@proxy.turnoxy.com:1318

# Omit pool = residential
sub_k8mX2pL9-country-US:pass@proxy.turnoxy.com:1318
```

## Combining with Other Parameters

```bash
# US datacenter IPs
sub_k8mX2pL9-pool-dc-country-US:pass@proxy.turnoxy.com:1318

# Japan mobile IPs with sticky session
sub_k8mX2pL9-pool-mob-country-JP-session-abc123:pass@proxy.turnoxy.com:1318
```

## Choosing the Right Pool

<AccordionGroup>
  <Accordion title="When to use Residential (res)">
    - Accessing sites with strict anti-bot measures
    - Social media automation
    - E-commerce operations
    - When you need IPs that look like real users
  </Accordion>

  <Accordion title="When to use Datacenter (dc)">
    - High-volume web scraping
    - Speed is more important than stealth
    - Targets with minimal anti-bot protection
    - Cost-sensitive operations
  </Accordion>

  <Accordion title="When to use ISP (isp)">
    - Long-running sessions
    - Balance between speed and quality
    - SEO monitoring
    - Ad verification
  </Accordion>

  <Accordion title="When to use Mobile (mob)">
    - Mobile app testing
    - Accessing mobile-only content
    - Maximum anonymity requirements
    - Bypassing sophisticated detection
  </Accordion>
</AccordionGroup>

## Errors

| Error Code | HTTP | Cause |
|------------|------|-------|
| `INVALID_POOL` | 400 | Value not `res`, `dc`, `isp`, or `mob` |
