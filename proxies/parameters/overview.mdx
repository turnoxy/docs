title: "Parameters Overview"

---

Parameters let you control IP pool type, geolocation, session behavior, and more.

## Format

Parameters are appended to the username using `-` as delimiter:

```
{username}-{param}-{value}-{param}-{value}:{password}
```

### Example

```bash
sub_k8mX2pL9-country-US-pool-dc-session-abc123:Qw4nR7vB3xY9mK2p@proxy.turnoxy.com:1318
```

## Available Parameters

| Parameter | Type | Values | Default | Description |
|-----------|------|--------|---------|-------------|
| [`pool`](/proxies/parameters/pool) | enum | `res`, `dc`, `isp`, `mob` | `res` | IP pool type |
| [`country`](/proxies/parameters/country) | string | ISO 3166-1 alpha-2 | random | Country code |
| [`state`](/proxies/parameters/state) | string | lowercase + underscore | - | State/province |
| [`city`](/proxies/parameters/city) | string | lowercase + underscore | - | City name |
| [`session`](/proxies/parameters/session) | string | 4-20 alphanumeric | - | Session ID |
| [`ttl`](/proxies/parameters/ttl) | integer | 1-60 | 30 | Session TTL (minutes) |
| [`timeout`](/proxies/parameters/timeout) | integer | 1-120 | 30 | Connection timeout (seconds) |

## Syntax Rules

| Rule | Description | Example |
|------|-------------|---------|
| Delimiter | `-` (dash) | `country-US-pool-dc` |
| Order | Not required | `pool-dc-country-US` = `country-US-pool-dc` |
| Param names | Case insensitive | `Country` → `country` |
| Duplicates | Last value wins | `country-US-country-JP` → `JP` |
| Unknown params | Ignored | Forward compatible |

## Quick Examples

<CodeGroup>
```bash US Residential
sub_k8mX2pL9-country-US:pass@proxy.turnoxy.com:1318
```

```bash Germany Datacenter
sub_k8mX2pL9-pool-dc-country-DE:pass@proxy.turnoxy.com:1318
```

```bash Sticky Session (10 min)
sub_k8mX2pL9-session-order123-ttl-10:pass@proxy.turnoxy.com:1318
```

```bash Full Example
sub_k8mX2pL9-pool-dc-country-US-state-california-city-los_angeles-session-task01-ttl-15:pass@proxy.turnoxy.com:1318
```
</CodeGroup>

## Validation Errors

Invalid parameter formats return HTTP `400 Bad Request`:

| Error | Cause |
|-------|-------|
| `LEADING_DASH` | Parameters start with `-` |
| `TRAILING_DASH` | Parameters end with `-` |
| `CONSECUTIVE_DASH` | Contains `--` |
| `INCOMPLETE_PAIR` | Missing parameter value |
| `TOO_MANY_PARAMS` | More than 10 parameters |

See [Error Reference](/errors) for the complete list.
