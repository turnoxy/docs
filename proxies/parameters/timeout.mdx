---
title: "timeout"
description: "Set connection timeout for proxy requests"
---

The `timeout` parameter controls how long the proxy waits when connecting to the target server.

## Specification

| Property | Value |
|----------|-------|
| Type | Integer |
| Range | 1-120 |
| Unit | Seconds |
| Default | 30 |

## Usage

```bash
# 60 second timeout
sub_k8mX2pL9-timeout-60:pass@proxy.turnoxy.com:1318

# 10 second timeout (faster failures)
sub_k8mX2pL9-country-US-timeout-10:pass@proxy.turnoxy.com:1318

# 120 second timeout (maximum, for slow targets)
sub_k8mX2pL9-timeout-120:pass@proxy.turnoxy.com:1318
```

## What It Controls

The timeout applies to the **proxy → target server** connection:

```
Client ──→ Proxy ══timeout══→ Target Server
                     ↑
              This connection
```

If the proxy cannot establish a connection to the target within the timeout period, the request fails.

## Choosing Timeout Values

| Timeout | Use Case |
|---------|----------|
| 5-10s | Fast APIs, well-optimized targets |
| 30s | Standard web requests (default) |
| 60s | Slower servers, heavy pages |
| 120s | Very slow targets, large file downloads |

<Note>
This timeout only controls the initial connection. Once connected, data transfer is not affected by this setting.
</Note>

## Combining with Other Parameters

```bash
# US datacenter with 60s timeout
sub_k8mX2pL9-pool-dc-country-US-timeout-60:pass@proxy.turnoxy.com:1318

# Sticky session with short timeout
sub_k8mX2pL9-session-task01-timeout-15:pass@proxy.turnoxy.com:1318
```

## Errors

| Error Code | HTTP | Cause |
|------------|------|-------|
| `INVALID_TIMEOUT` | 400 | Non-integer value |
| `TIMEOUT_OUT_OF_RANGE` | 400 | Value outside 1-120 |
