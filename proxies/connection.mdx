## Endpoints

| Protocol | Host | Port |
|----------|------|------|
| HTTP/HTTPS | `p1.turnoxy.com` | `1318` |

## URL Format

```
{protocol}://{username}-{params}:{password}@{host}:{port}
```

```
http://sub_k8mX2pL9-country-US:Qw4nR7vB3xY9mK2p@p1.turnoxy.com:1318
└─┬─┘ └────┬─────┘ └────┬────┘ └───────┬────────┘ └──────┬───────┘ └┬─┘
  │        │            │              │                 │          │
protocol username   parameters      password           host       port
```

## HTTP/HTTPS

```bash
# HTTP
curl -x "http://sub_k8mX2pL9:Qw4nR7vB3xY9mK2p@p1.turnoxy.com:1318" \
     http://example.com

# HTTPS (same proxy URL, different target)
curl -x "http://sub_k8mX2pL9:Qw4nR7vB3xY9mK2p@p1.turnoxy.com:1318" \
     https://example.com
```

<Note>
The proxy URL uses `http://` even for HTTPS targets. The proxy creates an encrypted CONNECT tunnel.
</Note>
